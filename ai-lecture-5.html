🧭 What Makes Local Search Different?

Fundamental Insight: Local search abandons the systematic exploration approach of traditional algorithms in favor of efficiency and practicality.

Traditional Search (BFS, DFS, A*)

  • Maintains frontier of unexplored states
  • Guarantees optimal solution if one exists
  • Memory usage: O(b^d) where b=branching factor, d=depth
  • Time: Exhaustive in worst case

Local Search

  • Maintains only current state
  • No guarantee of optimality
  • Memory usage: O(1)
  • Time: Often finds "good enough" solution quickly
Key Trade-off: We sacrifice completeness and optimality for efficiency and scalability

Common Misconception

Local search is not about "searching locally" in a geographic sense. It's about making decisions based only on the immediate neighborhood of the current solution state.

📊 Mathematical Foundation

Optimization Problem:
Find x* such that f(x*) = max{f(x) : x ∈ S}

Where:

  • S = State space (all possible configurations)
  • f(x) = Objective function (what we're optimizing)
  • N(x) = Neighborhood function (states reachable in one step)
  • x* = Global optimum (best possible solution)

Example: 8-Queens Problem

S: All possible placements of 8 queens on 8×8 board = 64C8 ≈ 4.4 × 10^9 states

Objective Function

f(x): 28 - (number of attacking pairs), where 28 = 8C2 (max non-attacking pairs)

Neighborhood

N(x): All states reachable by moving one queen to different row in same column = 7×8 = 56 neighbors

Global Optimum

x*: Any configuration where f(x*) = 28 (no attacking pairs)

Understanding Every Detail

🔬 Hill Climbing Algorithm: Complete Analysis

function HILL-CLIMBING(problem):
  current ← MAKE-NODE(INITIAL-STATE(problem))
  loop do:
    neighbor ← highest-valued successor of current
    if VALUE(neighbor) ≤ VALUE(current):
      return STATE(current)
    current ← neighbor

Complexity Analysis:

Metric Basic Hill Climbing Notes
Time Complexity O(n × |N|) n = steps to convergence, |N| = neighborhood size
Space Complexity O(1) Only stores current state
Completeness No Can get stuck in local optima
Optimality No Local optimum ≠ global optimum

Critical Implementation Detail

The algorithm's performance heavily depends on the neighborhood function choice. A poorly designed neighborhood can make the search space disconnected!

⚠️ The Three Fundamental Problems

Why Hill Climbing Gets Stuck

🚧 Local Optima: The Mathematics of Getting Trapped

Local Maximum:
∀x' ∈ N(x): f(x') ≤ f(x)
Global Maximum:
∀x' ∈ S: f(x') ≤ f(x*)

1. Local Maxima

Problem: Algorithm terminates at a peak that isn't the highest globally.

Real Example: 8-Queens

State with h=1 (only 1 attacking pair) but not h=0 (solution)

Probability Analysis

P(success) = 0.14 for random start in 8-Queens

86% of starts lead to local optima!

2. Plateaus (Flat Local Maxima)

Shoulder Problem

Algorithm wanders aimlessly on flat regions where all neighbors have same value. Can loop infinitely without progress detection.

3. Ridges

Issue: Optimal path requires diagonal movement, but algorithm can only move along coordinate axes.

🎲 Random Restarts: Mathematical Guarantee

Expected Number of Restarts:
E[Restarts] = 1/p
Expected Total Steps:
E[Steps] = E[Steps per trial] × E[Restarts]

8-Queens Calculation

  • Success probability: p = 0.14
  • Average steps per trial: 4
  • Expected restarts: 1/0.14 ≈ 7.14
  • Expected total steps: 4 × 7.14 ≈ 28.6

Probability of Failure

P(failure after k restarts) = (1-p)^k = (0.86)^k

Optimality Analysis

Theorem: Random-restart hill climbing is complete for finite state spaces.

Proof: P(never finding solution) = lim(k→∞) (1-p)^k = 0 when p > 0

Practical Result: 99.98% success rate with just 50 restarts, average 200 total steps for 8-Queens. This is incredibly efficient compared to exhaustive search of 4.4 billion states!

Trade-off Analysis

While success rate improves dramatically (94% vs 14%), the algorithm becomes 5x slower per attempt. The net effect is still positive: expected steps = 0.94 × 21 + 0.06 × 64 ≈ 23.6 vs original 4 steps, but success probability is much higher.

Plateau Navigation Strategy

When on a plateau, the algorithm performs a "random walk" among equal-valued states, hoping to find a slope that leads upward.

Infinite Loop Prevention

The sideways move limit prevents infinite loops on large plateaus. The limit of 100 is empirically determined for 8-Queens but should be adjusted for other problems.

🌡️ Simulated Annealing: Physics Meets AI

function SIMULATED-ANNEALING(problem, schedule):
  current ← MAKE-NODE(INITIAL-STATE(problem))
  for t = 1 to ∞:
    T ← schedule(t)
    if T = 0: return current
    next ← random successor of current
    ΔE ← VALUE(next) - VALUE(current)
    if ΔE > 0: current ← next
    else: current ← next with probability e^(ΔE/T)

The Physics Connection:

Metallurgical Annealing

Heat metal → atoms move randomly → slowly cool → atoms settle into low-energy crystalline structure

Boltzmann Distribution

P(state) ∝ e^(-E/kT) where E=energy, k=Boltzmann constant, T=temperature

Higher energy states become less probable as temperature decreases

AI Adaptation

State energy = -f(x) (negative because we maximize f)

Accept worse moves with probability e^(ΔE/T)

Temperature Schedules:

Schedule Type Formula Convergence Quality
Linear T(t) = T₀ - αt Fast Poor
Exponential T(t) = T₀ × α^t Medium Good
Logarithmic T(t) = c/log(t) Very Slow Optimal*
Acceptance Probability Examples (ΔE = -2):
T=10: P = e^(-2/10) = 0.82
T=1: P = e^(-2/1) = 0.14
T=0.1: P = e^(-2/0.1) = 0.0000003
*Theoretical guarantee: Logarithmic cooling ensures convergence to global optimum, but impractically slow. Exponential cooling with α=0.95 is most common in practice.

⚙️ Implementation Best Practices

Critical Design Decisions:

1. Neighborhood Design

  • Too small: Gets stuck easily (1-opt for TSP)
  • Too large: Expensive to evaluate (n-opt for TSP)
  • Sweet spot: 2-opt or 3-opt for TSP, single-queen moves for 8-Queens

2. Initial State Selection

  • Random: Simple, unbiased
  • Greedy construction: Start closer to optimum
  • Problem-specific heuristics: Domain knowledge

3. Termination Conditions

  • Solution found: Objective reached
  • Time limit: Real-time constraints
  • Iteration limit: Computational budget
  • No improvement: Convergence detection

4. Performance Monitoring

  • Track best solution found so far
  • Monitor convergence rate
  • Detect premature convergence
  • Log restart frequency and reasons

function robustHillClimbing(problem, config) {
  bestSolution = null
  attempts = 0
  while (!terminationCondition()) {
    current = generateInitialState()
    solution = basicHillClimbing(current)
    if (solution.quality > bestSolution.quality) {
      bestSolution = solution
    }
    attempts++
  }
  return bestSolution
}

🎯 Complete Mastery Summary

The Local Search Philosophy: "Perfect is the enemy of good" - find excellent solutions efficiently rather than optimal solutions expensively.

Algorithm Selection Guide:

Use Basic Hill Climbing When:

  • You need a quick approximation
  • Memory is extremely limited
  • Landscape has few local optima

Use Random Restarts When:

  • Solution quality matters more than speed
  • Many local optima present
  • You want reliability and simplicity

Use Simulated Annealing When:

  • Continuous optimization problems
  • You can't restart easily
  • Willing to tune temperature schedule

Use Genetic Algorithms When:

  • Complex representation spaces
  • Want to explore multiple solutions simultaneously
  • Have time for parameter tuning
